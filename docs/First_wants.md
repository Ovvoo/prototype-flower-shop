Техническое задание на создание сайта-каталога для цветочного салона

1. Введение

1.1. Назначение документа

Настоящее техническое задание описывает требования к разработке сайта-каталога для цветочного салона "Название салона". Цель проекта — создание современного, удобного и функционального онлайн-каталога товаров и услуг с возможностью оформления заказов.

1.2. Цели проекта

· Предоставление клиентам информации о товарах и услугах салона
· Увеличение продаж через онлайн-канал
· Автоматизация процесса приема заказов

2. Общее описание проекта

2.1. Целевая аудитория

· Клиенты салона в возрасте 25-55 лет
· Корпоративные клиенты
· Пользователи с разным уровнем компьютерной грамотности

2.2. Основные функции

· Каталог товаров с фильтрацией
· Система оформления заказов
· Управление контентом (CMS)
· Интеграция с социальными сетями

3. Требования к функционалу

3.1. Публичная часть (Frontend)

3.1.1. Главная страница

· Слайдер с акциями и новинками
· Категории товаров (букеты, комнатные растения, подарочные наборы и т.д.)
· Блок "Популярные товары"
· Информация о компании
· Отзывы клиентов
· Контактная информация

3.1.2. Каталог товаров

· Древовидная структура категорий
· Карточки товаров с фото, названием, ценой
· Расширенный фильтр:
  · По категориям
  · По цене
  · По типу цветов
  · По поводу (свадьба, день рождения, юбилей и т.д.)
  · По цветовой гамме
· Сортировка: по популярности, цене, новизне
· Быстрый просмотр товара
· Пагинация

3.1.3. Карточка товара

· Галерея изображений (3-5 фото)
· Название, описание
· Цена, наличие
· Варианты оформления (упаковка, открытка)
· Похожие товары
· Отзывы о товаре
· Кнопка "Добавить в корзину"
· Кнопка "Быстрый заказ"

3.1.4. Корзина и оформление заказа

· Корзина с возможностью изменения количества
· Форма оформления заказа:
  · Контактные данные (ФИО, телефон, email)
  · Адрес доставки с выбором из списка городов
  · Дата и время доставки (календарь с выбором времени)
  · Способ оплаты (онлайн, при получении)
  · Пожелания к заказу
· Расчет стоимости с учетом доставки
· Возможность применения промокодов

3.1.5. Дополнительные страницы

· О компании
· Услуги (оформление мероприятий, подписка на цветы и т.д.)
· Советы по уходу
· Контакты с картой проезда
· Блог/новости
· Акции и скидки

3.2. Административная часть (Backend)

3.2.1. Управление товарами

· CRUD-операции для товаров
· Загрузка нескольких изображений
· Управление категориями и подкатегориями
· Управление ценами, скидками
· Управление наличием товаров

3.2.2. Управление заказами

· Просмотр списка заказов
· Изменение статусов заказов
· Фильтрация заказов
· Экспорт заказов в Excel
· История изменений заказа

3.2.3. Контент-менеджмент

· Управление страницами сайта
· Публикация новостей и акций
· Управление отзывами
· Редактор текстов с WYSIWYG

3.2.4. Аналитика и отчеты

· Статистика продаж
· Популярные товары
· Конверсия заказов

3.3. Интеграции

· Онлайн-оплата (ЮKassa, Robokassa)
· СМС-уведомления
· Email-рассылка
· Интеграция с 1С (при необходимости)
· API для мобильного приложения (на будущее)

4. Технические требования

4.1. Технологический стек

· Frontend: HTML5, CSS3, JavaScript (React/Vue.js), адаптивная верстка
· Backend: PHP 8.0+ (Laravel) или Python (Django)
· База данных: MySQL 8.0+ или PostgreSQL
· Сервер: Nginx/Apache
· Хостинг: поддержка SSL, PHP, MySQL

4.2. Требования к производительности

· Время загрузки страницы: не более 2 секунд
· Оптимизация изображений
· Кэширование страниц
· Минификация CSS/JS

4.3. Безопасность

· Защита от SQL-инъекций
· CSRF-токены


4.4. Адаптивность и кроссбраузерность
· Минимальное разрешение: 320px

5. Дизайн и пользовательский интерфейс

· Цветовая гамма: нежные пастельные тона, акценты в зеленых и розовых оттенках
· Современный минималистичный дизайн
· Удобная навигация
· Высококачественные изображения товаров


· Интуитивно понятный интерфейс
· Минимальное количество кликов до заказа
· Четкие призывы к действию
· Простая форма заказа

6. Требования к контенту
· Описание каждого товара (100-200 символов)
· SEO-оптимизированные заголовки и мета-теги
· Уникальные тексты для страниц

6.2. Изображения

· Минимум 3 фото для каждого товара
· Разрешение: от 800x800px
· Формат: WebP с fallback на JPEG
· Вес: не более 200 КБ после оптимизации

7. SEO-требования

7.1. Базовая SEO-оптимизация

· ЧПУ (человеко-понятные URL)
· Настройка заголовков H1-H6
· Оптимизация мета-тегов
· Карта сайта XML
· Файл robots.txt

7.2. Техническое SEO

· Мобильная адаптация
· Высокая скорость загрузки
· Структурированные данные (Schema.org)

8. Этапы разработки


· Анализ конкурентов
· Прототипирование
· Разработка дизайна


· Функциональное тестирование
 Наполнение контентом

Все функции из ТЗ реализованы
· Нет критических ошибок
· Сайт работает на основных браузерах и устройствах
